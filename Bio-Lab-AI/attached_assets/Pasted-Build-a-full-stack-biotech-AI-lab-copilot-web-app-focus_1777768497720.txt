Build a full-stack biotech AI lab copilot web app focused on **Synergy H1 + Gen5** plate-reader data.

1) upload Synergy H1 / Gen5 plate data (CSV/Excel),

STAGE 1 – BACKEND + DB (core experiment model)

Create a Flask backend with:

SQLite schema “experiments” containing at least:
- id (int, PK)
- name (text)
- date (text or date)
- assay_type (text, e.g., “viability”)
- instrument (text, default “Synergy H1”)
- raw_file_name (text, nullable)
- raw_file_content (text or blob)
- plate_summary_json (text: JSON with computed stats and plate info)
- notes (text)
- status (text: “success” / “failed” / “unknown”)
- ai_summary (text)
- ai_next_experiments_json (text: JSON)
- created_at (timestamp)
- updated_at (timestamp)

API endpoints:
- GET /api/experiments → list experiments (id, name, date, status, assay_type)
- GET /api/experiments/<id> → full details including plate_summary_json
- POST /api/experiments → create new experiment with metadata + file
- PUT /api/experiments/<id> → update fields like status, notes, etc.
- Optional DELETE /api/experiments/<id>

File handling:
- Accept CSV and Excel (.csv, .xls, .xlsx).
- For now, store raw file content (or path) and parse immediately into structured data.
- Implement basic validation and return clear error messages on bad uploads.

STAGE 2 – PLATE DATA PARSING FOR SYNERGY H1 / GEN5

Assume a Gen5-style CSV with columns like:
- Well (e.g., “A1”)
- SampleID
- Condition (e.g., “Control”, “DrugA”)
- Dose_uM (float or 0 for control)
- Signal (float)

Implement a parser that:
- Identifies control wells:
  - Condition == “Control” OR Dose_uM == 0
- Groups wells by dose:
  - Group rows by Dose_uM for treated wells
- Computes, for each dose group (+ control):
  - n wells
  - mean Signal
  - standard deviation of Signal
  - percent change vs control (if control exists)
- Flags obvious outliers (e.g., wells > 2.5 SD from group mean)

Store in plate_summary_json:
- control_stats: {mean, sd, n}
- per_dose_stats: list of
  - dose_uM
  - mean_signal
  - sd_signal
  - n
  - percent_change_vs_control
- outlier_wells: list of wells
- any other useful metadata

The app must handle:
- Missing Condition or Dose_uM columns (fallback to basic summary).
- Non-standard column names using a simple mapping or helpful error.

STAGE 4 – COPILOT (AI CHAT)

Add a chatbot panel tied to the **selected experiment**.

Backend:
- POST /api/chat:
  - Input: experiment_id, user_message
  - Backend fetches experiment by id
  - Fetches 2–3 most recent related experiments (same assay_type and instrument)
  - Builds a prompt for Gemini with:
    - System instruction
    - Current experiment summary (including Synergy H1 metrics)
    - Short summaries of related experiments (if any)
    - The user’s question

SYSTEM PROMPT (server-side, not exposed to user):

“You are an expert cell biologist and lab scientist copilot. You specialize in analyzing Synergy H1 / Gen5 plate-reader experiments. You help scientists diagnose failed or surprising experiments, identify patterns across multiple Synergy H1 runs, and suggest concrete next experiments. Always be specific, cite the numeric data you see (e.g., dose, mean signal, percent change vs control), compare with previous experiments if available, and give actionable recommendations. If the data is limited, state your uncertainty clearly and suggest what additional experiment would reduce uncertainty.”

User asks questions like:
- “Why is viability so low at 1 µM?”
- “How does this plate compare to my last Synergy H1 run?”
- “What doses should I test next if I want less toxicity?”

The API returns the AI text, and:
- Optionally a short summary and 3 structured suggestions.

Frontend chat UI:
- Chat window showing user and AI messages
- Input box + Send button
- Loading indicator when waiting for AI
- Display the AI answer in a readable, structured way
- Keep chat history for the current experiment in state

If possible, implement **streaming** from Flask to React (SSE or chunked), but if that’s too complex, a standard single-response is OK for now.
